## Use Cases

## Модель: Участник события

### 1. Регистрация участия пользователя в событии

**Цель**: Зарегистрировать участника на событие.

**Актор**: Пользователь.

**Описание**:

- Пользователь вводит свои данные (имя, фамилия, дата рождения, email) и выбирает событие.
- Система проверяет, что в событии есть свободные места.
- Если места есть, система добавляет пользователя в список участников события.

**Предусловия**:

- Пользователь не зарегистрирован на данное событие.
- Количество участников события меньше максимального.

**Результат**: Пользователь успешно зарегистрирован на событие.

---

### 2. Получение списка участников события

**Цель**: Получить список всех участников конкретного события.

**Актор**: Администратор / Пользователь.

**Описание**:

- Пользователь или администратор выбирает событие и запрашивает список участников.
- Система возвращает список всех зарегистрированных участников.

**Предусловия**:

- Событие существует.

**Результат**: Получен список участников события.

---

### 3. Получение определенного участника по его Id

**Цель**: Получить информацию об участнике по его идентификатору.

**Актор**: Администратор / Пользователь.

**Описание**:

- Пользователь или администратор вводит Id участника.
- Система возвращает информацию о данном участнике.

**Предусловия**:

- Участник зарегистрирован на событие.

**Результат**: Получена информация об участнике.

---

### 4. Отмена участия пользователя в событии

**Цель**: Удалить участника из списка события.

**Актор**: Пользователь / Администратор.

**Описание**:

- Пользователь или администратор выбирает событие и отменяет участие определенного участника.
- Система удаляет участника из списка участников.

**Предусловия**:

- Участник зарегистрирован на событие.

**Результат**: Участник удален из списка.

---

## Модель: Событие

### 1. Получение списка всех событий

**Цель**: Получить все существующие события.

**Актор**: Пользователь.

**Описание**:

- Пользователь отправляет запрос для получения списка всех событий.
- Система возвращает список всех доступных событий.

**Предусловия**:

- События существуют в базе данных.

**Результат**: Получен список всех событий.

---

### 2. Получение определенного события по его Id

**Цель**: Получить информацию о событии по его идентификатору.

**Актор**: Пользователь / Администратор.

**Описание**:

- Пользователь или администратор вводит Id события.
- Система возвращает информацию о данном событии.

**Предусловия**:

- Событие существует.

**Результат**: Получена информация о событии.

---

### 3. Получение события по его названию

**Цель**: Найти событие по его названию.

**Актор**: Пользователь / Администратор.

**Описание**:

- Пользователь или администратор вводит название события.
- Система возвращает информацию о событии с таким названием.

**Предусловия**:

- Событие с таким названием существует.

**Результат**: Найдено событие по названию.

---

### 4. Добавление нового события

**Цель**: Создать новое событие.

**Актор**: Администратор.

**Описание**:

- Администратор вводит данные нового события (название, описание, дата, место, категория, максимальное количество участников).

**Предусловия**:

- У администратора есть доступ к созданию событий.

**Результат**: Событие добавлено.

---

### 5. Изменение информации о существующем событии

**Цель**: Изменить информацию о уже существующем событии.

**Актор**: Администратор.

**Описание**:

- Администратор выбирает событие и изменяет его данные (например, дату, описание или категорию).
- Система обновляет информацию о событии в базе данных.

**Предусловия**:

- Событие существует.
- У администратора есть доступ к изменению событий.

**Результат**: Информация о событии обновлена.

---

### 6. Удаление события

**Цель**: Удалить событие.

**Актор**: Администратор.

**Описание**:

- Администратор выбирает событие для удаления.
- Система удаляет событие из базы данных.

**Предусловия**:

- Событие существует.

**Результат**: Событие удалено.

---

### 7. Получение списка событий по критериям

**Цель**: Найти события, соответствующие определенным критериям (дата, место, категория).

**Актор**: Пользователь.

**Описание**:

- Пользователь вводит критерии (дата, место, категория).
- Система возвращает список событий, соответствующих этим критериям.

**Предусловия**:

- События, соответствующие этим критериям, существуют.

**Результат**: Найден список событий по критериям.

---

### 8. Добавление изображений к событиям

**Цель**: Добавить изображение к событию.

**Актор**: Администратор.

**Описание**:

- Администратор загружает изображение для определенного события.
- Система сохраняет изображение и привязывает его к событию.

**Предусловия**:

- Событие существует.

**Результат**: Изображение добавлено к событию.
